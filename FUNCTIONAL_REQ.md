# Functional Requirements

# 1. Project Creation & Management  
Requirement  
- The system shall allow project managers to create, edit, and delete projects.  
Acceptance Criteria  
- Project details (title, deadline, description) must be editable.  
- Only project managers can create projects.  

# 2. Task Assignment & Tracking  
Requirement  
- The system shall allow project managers to assign tasks to team members.  
Acceptance Criteria  
- Assigned tasks must include priority levels (High, Medium, Low).  
- Team members should receive task notifications.  

# 3. Deadline Notifications  
Requirement  
- The system shall send automated notifications for approaching task deadlines.
Acceptance Criteria  
- Users should receive reminders via email and in-app notifications.  
- Notifications should be sent 24 hours before the deadline.  

# 4. User Authentication  
Requirement  
- The system shall require users to log in using email and password authentication.  
Acceptance Criteria
- Passwords must be encrypted using AES-256.  
- Users must reset passwords via email verification.  

# 5. Reporting & Analytics  
Requirement
- The system shall generate reports on project progress, delays, and workload distribution.  
Acceptance Criteria  
- Reports should be downloadable as PDF and CSV formats.  
- Managers should be able to filter reports by team member, project, and status.
